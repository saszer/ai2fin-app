# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AI2 Financial Platform, we appreciate your help in disclosing it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: **security@ai2fin.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

### What to Expect

- **Acknowledgement**: Within 48 hours of your report
- **Assessment**: We will evaluate the severity and impact within 5 business days
- **Resolution**: Critical vulnerabilities are prioritized for immediate patching
- **Disclosure**: We coordinate disclosure timing with the reporter

### Scope

In scope:
- The AI2 Financial Platform application and its APIs
- Authentication and authorization mechanisms
- Data handling and storage practices
- Third-party integration security

Out of scope:
- Social engineering attacks
- Denial of service attacks
- Issues in third-party dependencies (report these to the upstream project)
- Issues that require physical access to a user's device

### Safe Harbor

We will not pursue legal action against researchers who:
- Make a good faith effort to avoid privacy violations and data destruction
- Only interact with accounts they own or with explicit permission
- Do not exploit a vulnerability beyond what is necessary to confirm it
- Report findings promptly and do not disclose publicly before a fix is available

### Recognition

We acknowledge security researchers who report valid vulnerabilities. With your permission, we will credit you in our security advisories.
